## UI 튜닝 작업 핵심 정리

### 1. 그래프 탭 (`GraphTab.vue`, `NodeDetailPanel.vue`, `NvlGraph.vue`)
- **노드 패널**
  - 우측 플로팅 패널 상단 고정, 콘텐츠 높이에 맞는 `min-height`/`max-height` 적용.
  - 패널 토글 버튼을 그래프/UM L 영역 내부 우측 상단에 정사각형 버튼으로 배치, 패널이 열리면 버튼 숨김.
  - 패널/스텝 패널 그림자, 보더 컬러를 옅게 조정해 과한 입체감 제거.
  - 패널 리사이즈 핸들 공통 믹스인(`useResize`) 적용으로 가로 리사이즈 지원.

- **콘솔**
  - 그래프/전환 탭 모두 콘솔을 하단 고정 플로팅 패널로 통일, 세로 리사이즈 지원.
  - 로그가 생기면 자동으로 콘솔 열림, 닫혀 있을 때만 중앙 하단 토글 버튼 표시.
  - 콘솔 내부에 헤더(제목+개수)와 하단 중앙 `▼` 닫기 버튼 분리.

- **노드/관계 표시 & limit**
  - 하단 요약 텍스트를 영어로 통일: `Displaying X nodes, Y relationships.`.
  - 배경색, 불필요한 라인 제거 후 다른 요소와 좌측 정렬 및 여백만으로 구분.
  - limit 적용 시 "Displaying X nodes, Y relationships." 바로 아래에 영어 경고 메시지 표시:
    - "More than 500 nodes were found; only 500 are visible due to the display limit." (주황색 텍스트)
    - 이모지 없이 주황색(`#ea580c`) 텍스트만으로 경고 스타일 적용.
  - 그래프 상단 limit-notice 제거 (NvlGraph.vue의 왼쪽 상단 배지 삭제).
  - 노드 패널 스크롤 개선:
    - 이중 스크롤 구조 제거 (`.stats-wrapper`의 `overflow-y` 제거, `.panel-body`에서만 스크롤).
    - `.display-summary`의 `position: sticky` 제거하여 스크롤이 끝까지 도달하도록 수정.
    - `.floating-panel`에 `height: 100%`, `.panel-body`에 `flex: 1`, `min-height: 0` 적용.
    - 스크롤바 스타일 추가로 표시 여부 명확화.

### 2. 전환 탭 (`ConvertTab.vue`, `FrameworkSteps.vue`)
- **파일 탐색기**
  - 접기/펼치기 제거, 좌측 패널 가로 리사이즈만 가능하게 변경.
  - 초기 너비/최소/최대 너비를 상향 조정해 더 넓게 보이도록 개선.

- **단계(FrameworkSteps)**
  - 불필요한 레이어 제거 후, 한 줄 리스트 형태로 단순화.
  - 상태별 배경색·텍스트 색상·숫자 배지 스타일 재정의 및 hover, pulse, glow 효과로 시각적 피드백 강화.

- **탭 아이콘**
  - 상단 로고: `⚡ Robo Analyzer`로 통일.
  - 메인 탭: 업로드(📁), 그래프(📊), 전환(🔀)로 서로 다른 아이콘 사용.

### 3. 업로드 탭 (`UploadTab.vue`, `DropZone.vue`)
- **빈 상태 가이드**
  - 우측 뷰 영역에 업로드 가이드를 카드 형태로 배치, 간결한 4단계 흐름으로 정리.
  - 설명 텍스트를 짧고 직관적으로 재작성, 과도한 설명 제거.
  - 가이드 상자와 업로드 박스를 flex로 반반 차지하게 하고, 여백만으로 구분.

- **레이아웃 & 스크롤**
  - 우측 뷰는 항상 중앙 정렬, 높이 `100%`로 설정해 작은 화면에서도 정렬 유지.
  - 가이드 내부에만 얇은 커스텀 스크롤바를 적용해, 모양은 정제하되 영역은 고정.

### 4. 설정 모달 (`SettingsModal.vue`)
- **오버레이 & 레이아웃**
  - 배경 흐림 제거, 연한 어두운 오버레이(`rgba(0,0,0,0.1)`)만 사용.
  - 모달 크기를 `700x600` 고정하고 내부 콘텐츠에만 스크롤 적용.

- **탭/섹션 구조**
  - 좌측 탭: `표시 설정 / API 설정 / 세션 정보 / 데이터 관리`로 구성.
  - 활성 탭은 연한 배경+왼쪽 보더로만 표현, 파란/빨간 풀컬러 하이라이트 제거.
  - `데이터 관리` 제목은 일반 텍스트 색상으로, 강조는 설명과 버튼으로만 처리.

- **표시 설정**
  - 노드 표시 제한/UM L 깊이 슬라이더만 남기고, “현재 그래프: X 노드, Y 관계” 문구 제거.
  - 값 변경 시 자동으로 `localStorage` 및 상위 스토어에 반영, 별도 저장 버튼 제거.

- **API 설정**
  - API Key 입력은 `watch`로 세션 스토어와 자동 동기화, 저장 버튼 삭제.
  - 모달 열릴 때 세션 스토어의 API Key를 입력값으로 자동 로드.

- **세션 정보 & 데이터 관리**
  - 세션 정보에는 `Session ID`, `프로젝트명`만 표시.
  - 데이터 관리 섹션:
    - “새 세션 시작” 버튼은 중립 색상의 테두리 버튼.
    - “전체 삭제” 버튼은 텍스트만 빨간색인 테두리 버튼으로, hover 시 연한 붉은 배경으로만 경고.

### 5. 공통 UX 정리
- 기존 로직에 분기(옛 로직 보존용 if 등)를 추가하지 않고, **수정된 단일 로직**으로 정리.
- 리사이즈 핸들 스타일을 공통 믹스인으로 통합해 색·hover 효과를 일관되게 유지.
- 강조 색상은 대부분 중성 그레이 톤으로 맞추고, 진한 파랑/빨강은 필요한 곳(경고, 활성 상태)에만 최소 사용.

  - 상단 툴바/헤더가 차지하는 공간을 최소화하여 그래프, UML, 코드 영역이 더 넓게 보이도록 개선.
  - 노드 패널, 콘솔처럼 플로팅 패널 방식으로 전환 가능한 요소는 모두 플로팅으로 처리.

### 7. 그래프/UML 영역 개선
- **플로팅 패널 통합**
  - 노드 패널, UML 클래스 검색, 그래프/UML 전환 버튼을 모두 그래프 영역 내부에 플로팅 패널로 배치.
  - `>` `<` 버튼으로 간단하게 열고 닫을 수 있도록 제공.
  - 패널을 닫으면 그래프/UML 영역이 더 넓게 보이도록 구현.

- **그래프/UML 전환 UI**
  - 그래프 영역 내부에 "Graph" / "UML" 전환 버튼을 플로팅 형태로 배치.
  - 각 모드에서 검색창을 제공하여 노드/클래스 검색 가능.

- **그래프 상호작용 개선**
  - 더블클릭으로 노드 확장/축소 토글 기능 (이미 펼쳐진 노드는 다시 접기).
  - 상호작용 시 빠른 반응을 위한 최적화 (배치 렌더링, 가상화 등 고려).

### 8. UML 클래스 다이어그램 개선
- **VueFlow 기반 UML 생성**
  - Neo4j 데이터를 직접 활용하여 VueFlow로 UML 클래스 다이어그램 생성.
  - 기존 백엔드 요청 로직은 주석 처리하고, 프론트엔드에서 Neo4j 데이터 기반으로 처리.

- **깊이 조절 기능**
  - 설정에서 UML 깊이를 조절할 수 있는 UI 제공 (1~10단계).
  - 선택된 클래스 기준으로 설정된 깊이만큼만 관계를 표시.

- **클래스 확장 기능**
  - 특정 클래스를 클릭/더블클릭하여 확장 가능.
  - 노드 패널에서 "클래스 다이어그램 생성" 대신 "확장" 기능으로 변경.

- **DEPENDENCY 필터링 로직**
  - `is_value_object: true`인 DEPENDENCY 제외.
  - 상위 클래스에 소유 관계(COMPOSITION/AGGREGATION/ASSOCIATION)가 있으면 하위 클래스의 DEPENDENCY 제거.
  - 상속 체인을 거슬러 올라가 상위 클래스에 소유 관계가 있으면 하위 클래스의 DEPENDENCY 제거.

- **머메이드 스타일 표현**
  - UML 클래스 다이어그램을 머메이드 다이어그램 스타일로 표현.

### 9. 전환 탭 개선
- **단계 표시**
  - 상단 레이어에 단계를 배치하지 않고, 플로팅 패널이나 접기/펼치기 방식으로 제공.

  - 파일 클릭 시 해당 코드 영역이 에디터에 표시되도록 구현.

- **이름 변경**
  - "전환결과" → "전환"으로 탭 이름 변경.

### 12. 전체 UI 원칙
- **명확성**
  - 사용자가 "이게 뭐지?" 하고 방황하지 않도록 명확하고 확실한 UI 제공.
  - 모든 기능과 상태를 직관적으로 표현.

- **레거시 제거**
  - 레이아웃 수정 시 기존 레이아웃을 완전히 제거하고 새로 개편.
  - 기존 레거시 흔적을 남기지 않고 공존하지 않도록 처리.

- **색상 통일**
  - 전체적으로 밝은 색상 사용 (어두운 색상 제거).
  - 일관된 색상 팔레트 적용.

- **콘솔 높이 조절**
  - 콘솔 영역이 너무 많은 공간을 차지하지 않도록 높이 조절 가능하게 제공.
